01
Overview
Purpose and scope of this configuration guide

This document describes the end-to-end process for configuring Connxt V2 PSP Integration in a Production environment for an existing client. Complete all sections in order for a correct and verifiable setup.

  • Creating a Brand in the Connxt V2 Admin Portal
  • Enabling and configuring the PSP Integration API in the client database
  • Configuring the PSP through the CRM Wallet Configuration page
  • Updating all PaymentGatewayId references across related tables
  • Verifying the Wire Transfer Dynamic Bank Method configuration
i
All SQL queries are formatted for Microsoft SQL Server (T-SQL). Each query block includes a Copy button — click it to copy the exact query to your clipboard instantly.
02
Preconditions
Required access and credentials before starting

Confirm all of the following are available before proceeding:

  • Connxt V2 Admin Portal — admin-level access required
  • Client Production Database — write access to Settings and CRM tables
  • CRM Admin Panel — access to Wallet Configuration
  • Production PSP Merchant Credentials — provided by the payment provider
03
Create Brand in Connxt V2
Set up the client brand and generate the Secret Key
  1. Log in to the Connxt V2 Admin Portal.
  2. Navigate to Brand Management.
  3. Click Create New Brand and complete the client details.
  4. Set the Origin URL for the brand — e.g. https://clientdomain.com
!
Important: After saving the brand, Connxt automatically generates a Secret Key. Store this securely — it is required in Section 04 step 4.6 to configure the authentication key.

Section 04 · Enable PSP Integration in Client Database03
04
Enable PSP Integration in Client Database
Execute all queries against the Client Production Database → Settings table
4.1Enable PSP Integration API
settings.sqlT-SQL
UPDATE Settings
  SET   [Value] = 'true'
  WHERE [Name]  = 'IsPSPIntegerationAPIEnabled';
4.2Configure Base API URL
settings.sqlT-SQL
UPDATE Settings
  SET   [Value] = 'https://api.connect-prod.fynxt.com/connxt/v2'
  WHERE [Name]  = 'PSP Integeration API';
4.3Configure Add PSP Endpoint
settings.sqlT-SQL
UPDATE Settings
  SET   [Value] = '/psps/external'
  WHERE [Name]  = 'PSPIntegerationAPIAddPSP';
!
Important: If PSPIntegerationAPIAddPSP does not exist in the Settings table, use the INSERT in step 4.4 instead.
4.4Insert Setting — if row does not exist
settings.sqlT-SQL
INSERT INTO Settings
  (Name, Description, Value, DataType, IsForCRM, IsForCP, IsForCPMobile)
  VALUES
  ('PSPIntegerationAPIAddPSP', 'API URL for Add a PSP Integeration',
   '/psps/external', 'string', 1, 0, 0);
4.5Configure Transaction Endpoint
settings.sqlT-SQL
UPDATE Settings
  SET   [Value] = '/transactions'
  WHERE [Name]  = 'PSPIntegerationAPIoutboundinitiateURL';
4.6Configure Authentication Secret Key
settings.sqlT-SQL
UPDATE Settings
  SET   [Value] = '{SECRET_KEY}'
  WHERE [Name]  = 'PSPIntegerationAPIAuthKey';
i
Replace {SECRET_KEY} with the Secret Key generated when the brand was created in Section 03.

Section 05–06 · Configure PSP & Update References04
05
Configure PSP in Wallet Configuration Page
Set up production merchant details via CRM Admin Panel
  1. Log in to the CRM Admin Panel.
  2. Navigate to Wallet → Payment Gateway Configuration.
  3. Enter the Production Merchant Details for the PSP.
  4. Save the configuration.

Upon saving, records are created automatically in:

Table NameDescription
CRM_PaymentGatewayMasterPrimary payment gateway record
CRM_PaymentGatewayAPIMasterAPI configuration record
CRM_PaymentGatewayCurrencyCurrency mapping record
!
Important: A new PaymentGatewayId is generated automatically. Record this value — needed for all queries in Section 06. Example: Old ID = 66 → New ID = 200 (illustrative only).
06
Update Existing PSP References
Replace OLD_ID with previous PSP ID and NEW_ID with the newly generated ID
i

Verify Wire Transfer Dynamic Bank Method
Confirm configuration record in CRM_paymentgatewayCustomFields

Verify that the dynamic bank method configuration record exists in the CRM_paymentgatewayCustomFields table.

!
Important: Wire Transfer always uses PaymentGatewayId = 4. Only confirm the record exists — no UPDATE is needed if it is already present.
08
Final Validation Checklist
Confirm all steps are complete before signing off
  • Brand created in Connxt V2 Admin Portal
  • Origin URL configured for the brand
  • Secret Key generated and stored securely
  • All PSP Integration Settings updated in client database (4.1–4.6)
  • PSP configured via Wallet Configuration page in CRM Admin
  • New PaymentGatewayId generated and recorded
  • All 11 PSP reference tables updated with new PaymentGatewayId (6.1–6.11)
  • Wire Transfer dynamic config confirmed for PaymentGatewayId = 4
